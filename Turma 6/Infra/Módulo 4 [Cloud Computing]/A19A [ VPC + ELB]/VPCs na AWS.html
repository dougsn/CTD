VPCs na AWS

Funcionamento

A Amazon Virtual Private Cloud (VPC) oferece controle completo sobre seu ambiente de rede virtual, incluindo localização de recursos, conectividade e segurança. 

A primeira etapa é criar sua VPC. Os recursos podem ser adicionados, como instâncias do Amazon Elastic Compute Cloud (EC2) e Amazon Relational Database Service (RDS).

Por fim, você pode definir como suas VPCs se comunicam entre contas, zonas de disponibilidade (AZs) ou regiões. Nesse caso, o tráfego de rede é compartilhado entre duas VPCs em cada região.

Características da Amazon VPC

A Amazon Virtual Private Cloud oferece recursos que podem ser usados ​​para aumentar e melhorar a segurança de sua nuvem privada virtual (VPC):

Acessibilidade Analyzer: Esta é uma ferramenta de análise de configuração estática que permite analisar e depurar a acessibilidade de rede entre dois recursos em sua VPC. Após especificar os recursos de origem e destino na VPC, o analisador de acessibilidade produz detalhes hop-by-hop do caminho virtual entre eles quando são alcançáveis ​​e identifica o componente que causa uma falha quando não são alcançáveis.

Logs de fluxo de VPC: Você pode monitorar logs de fluxo de VPC entregues ao Amazon S3 ou Amazon CloudWatch para obter visibilidade operacional das dependências de rede e padrões de tráfego, detectar anomalias e evitar violações de dados, solucionar problemas de configuração e conectividade de rede. Os metadados ricos dos logs de fluxo ajudam a adquirir informações adicionais sobre quem iniciou as conexões TCP, bem como o destino e a origem reais no nível do pacote para o tráfego que flui pelas camadas intermediárias, como o gateway NAT. Você também pode arquivar logs de fluxo para ajudar a atender a determinados requisitos de conformidade.

Replicação de tráfego VPC: A replicação de tráfego VPC permite copiar o tráfego de rede de uma interface de rede elástica de instâncias do Amazon EC2 e, em seguida, enviar o tráfego para dispositivos de monitoramento e segurança fora de banda para inspeção profunda de pacotes. Com o espelhamento de tráfego de VPC, você pode detectar anomalias de rede e segurança, obter insights operacionais, implementar controles de segurança e conformidade e solucionar problemas. O espelhamento de tráfego da VPC é um recurso que fornece acesso direto aos pacotes de rede que passam pela VPC.

Roteamento de entrada: Isso permite direcionar todo o tráfego de entrada e saída que flui de/para um gateway da Internet (IGW) ou gateway privado virtual (VGW) para a interface de rede elástica de uma instância EC2 específica. Com esse recurso, uma nuvem privada virtual pode ser configurada para enviar todo o tráfego para uma instância IGW, VGW ou EC2 antes que o tráfego atinja as cargas de trabalho de negócios.

Grupos de segurança: Os grupos de segurança atuam como um firewall para instâncias do Amazon EC2 associadas. Eles controlam o tráfego de entrada e saída no nível da instância. Ao iniciar uma instância, ela pode ser associada a um ou mais grupos de segurança criados. Cada instância na VPC pode pertencer a um conjunto diferente de grupos de segurança. Se um security group não for especificado ao executar a instância, esta será automaticamente associada ao security group padrão da VPC.

Lista de controle de acesso à rede: Uma lista de controle de acesso à rede (ACL) é uma camada opcional de segurança para sua VPC que atua como um firewall para controlar o tráfego de entrada e saída de uma ou mais sub-redes. ACLs de rede com regras semelhantes a grupos de segurança podem ser configuradas para adicionar uma camada de segurança à sua VPC.

Usando outros recursos da AWS com Amazon VPC

Existem vários recursos que podem ser usados ​​com a Virtual Private Cloud (VPC):

  • AWS Transit Gateway: Conecte facilmente Amazon VPCs, contas da AWS e redes locais a um único gateway.

Ele otimiza a VPC e torna a infraestrutura facilmente escalável. Direciona todo o tráfego de cada VPC ou VPN e oferece um local para monitorar.

  • AWS Private Link: Usado para estabelecer conectividade privada entre VPCs e serviços hospedados na AWS ou no local, sem expor dados à Internet.
  • AWS Network Firewall: Permite implementar a segurança de rede em Amazon VPCs de forma simples e clara.
  • AWS VPN: Possibilita estender redes locais para a nuvem e acessá-las com segurança de qualquer lugar:

AWS Client VPN: é um serviço de VPN totalmente gerenciado e elástico que aumenta ou diminui automaticamente com base em seus requisitos. Por ser uma solução de VPN na nuvem, você não precisa instalar e gerenciar soluções baseadas em hardware ou software, nem tentar estimar quantos usuários remotos você pode dar suporte de uma só vez.

AWS Site-to-Site VPN: cria uma conexão segura entre seu data center ou filial e seus recursos da Nuvem AWS. Para aplicativos distribuídos globalmente, a opção Accelerated Site-to-Site VPN oferece desempenho ainda maior ao trabalhar com o AWS Global Accelerator.

  • Gateway de tradução de endereço de rede (NAT): fornece cargas de trabalho na sub-rede privada da VPC para acessar a Internet, evitando que a Internet inicialize uma conexão com essas instâncias.
